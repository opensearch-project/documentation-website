# OpenSearch Content Style Guide

Welcome to the OpenSearch Content Style Guide, a collection of guides that provide style, writing, and formatting standards for OpenSearch content.

| Guide | Describes | Applies to |
| :--- | :--- | :--- |
| [Style Guide](STYLE_GUIDE.md) | Naming conventions, voice and tone, and brand identity | All content |
| [Writing Guide](WRITING_GUIDE.md) | Grammar, punctuation, and word choice | All content |
| [Text Formatting Guide](TEXT_FORMATTING_GUIDE.md) | Text formatting, including code examples, lists, links, images, and procedures | Documentation, blog posts |
| [Markdown Formatting Guide](MARKDOWN_FORMATTING_GUIDE.md) | Markdown and Jekyll syntax | Documentation |
| [API Style Guide](API_STYLE_GUIDE.md) | API documentation structure | Documentation |
| [UI Style Guide](UI_STYLE_GUIDE.md) | OpenSearch Dashboards UI text | UI copy |
| [Blog Style Guide](BLOG_STYLE_GUIDE.md) | Blog post writing conventions | Blog posts |
| [Terms](TERMS.md) | Terminology and word usage | All content |

To start contributing, see [Contributing guidelines](https://github.com/opensearch-project/documentation-website/blob/main/CONTRIBUTING.md).
